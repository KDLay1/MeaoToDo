# Todo 日历 MVP 设计说明

## 目标

本分支实现 Todo 模块的日历显示能力。日历不是独立日程系统，而是 Todo 的另一种显示方式。

## 当前 MVP 范围

- 支持列表模式 / 日历模式切换。
- 日历模式支持本周、本月、年度三种视图。
- 本周视图采用时间表结构，按 00:00 - 23:00 分小时展示。
- 任务支持截止日期和具体截止时间。
- 使用 `hasDueTime` 区分“只有日期”和“日期 + 具体时间”。
- 只有日期、没有具体时间的任务显示在本周视图的“全天”行。
- 有具体时间的任务显示在对应小时格。
- 没有 `dueAt` 的任务不进入日历，只保留在列表/清单中。

## 数据语义

```text
TaskEntity.dueAt: Long?
TaskEntity.hasDueTime: Boolean
```

语义：

```text
只选择截止日期：
  dueAt = 当天 00:00
  hasDueTime = false

选择截止日期 + 截止时间：
  dueAt = 具体时间戳
  hasDueTime = true
```

## 本周视图

本周视图为 7 天 × 24 小时的时间表。

```text
顶部：本周范围 + 上一周 / 今天 / 下一周
表头：周一到周日
第一行：全天 / 无具体时间任务
主体：00:00 - 23:00 小时格
```

同一小时内有多个任务时，先显示前几项，超出用 `+N` 表示。

## 本月视图

本月视图显示月历网格。

- 日期格显示任务数量。
- 日期格显示少量任务标题。
- 点击某天后，下方显示当天任务列表。

## 年度视图

年度视图显示 12 个月概览。

- 显示每月任务总数。
- 显示每月未完成任务数。
- 点击月份后切换到本月视图。

## 暂不实现

以下内容不进入当前 MVP：

- 重复任务。
- 习惯追踪。
- 提醒通知。
- 拖拽修改日期。
- 多日任务。
- 任务持续时间。
- 系统日历同步。

## 后续目标：重复任务 / 习惯任务

后续可单独开分支实现：

```text
develop/todo-recurring
```

初步字段方向：

```text
repeatRule: String?
repeatEndAt: Long?
```

候选规则：

```text
none
daily
weekly
monthly
yearly
custom
```

重复任务需要单独设计：

- 单次完成与整组完成。
- 删除本次与删除全部。
- 日历展开实例。
- Wi-Fi 同步冲突处理。
- 未来任务实例生成策略。

因此当前分支只把它记录为后续目标，不落库、不生成重复实例。
